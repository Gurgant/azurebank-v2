## 1. Overview

This document contains ASCII wireframe specifications for all application views, covering both desktop and mobile layouts.

### Design Approach
- **Mobile-First**: Design for mobile, enhance for desktop
- **Responsive Breakpoints**: Mobile (0-479px), Tablet (480-1023px), Desktop (1024px+)

**Registration Form Fields**:
| Field | Type | Validation |
|-------|------|------------|
| First Name | `<Input>` | Required, 2-50 chars |
| Surname | `<Input>` | Required, 2-50 chars |
| **AzureTag** | `<Input prefix="@">` | Required, 3-20 chars, alphanumeric+_, unique, IMMUTABLE |
| Email | `<Input type="email">` | Required, valid format, unique |
| Password | `<Input type="password">` | Required, 8+ chars, complexity |
| Confirm Password | `<Input type="password">` | Must match password |

## 6. Transfer Wizard Component Summary

The External Transfer feature uses a multi-step wizard pattern:

| Step | Component | Key Elements |
|------|-----------|--------------|
| 1. Select Source | `<TransferSourceStep>` | RadioGroup for accounts |
| 2. Find Recipient | `<TransferRecipientStep>` | Search input, recipient card |
| 3. Enter Amount | `<TransferAmountStep>` | Amount input, quick buttons, description |
| 4. Confirm | `<TransferConfirmStep>` | Summary card, confirm button |
| Success | `<TransferSuccessStep>` | Checkmark, reference number |

**State Management**: Transfer wizard state stored in Redux slice `transferWizard`:
- `currentStep`: 1-4
- `sourceAccountNumber`: string
- `recipientAzureTag`: string
- `recipientAccountNumber`: string
- `amount`: number
- `description`: string
